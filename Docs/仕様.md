# GeoTagger 仕様書

## 概要

GeoTaggerは、GPXトラックログを使って写真（JPEG）にジオタグ（GPS座標）を付与するmacOSデスクトップアプリケーションです。
GeoShutterで記録したGPXファイルとの連携を主なユースケースとして設計します。

---

## 背景・目的

- GeoShutter（iPhoneアプリ）でGPXトラックログを記録する
- SONYカメラ（α1、α7RV等）で撮影した写真にGPXから座標を付与したい
- Lightroomのジオタグ機能はタイムゾーン処理が不安定で、誤った位置が付くことがある
- カメラのタイムゾーン設定（UTC-7/UTC-8等）がファイルによって異なる場合がある
- 旅行先によってはカメラのタイムゾーン設定と現地時間が一致しない場合がある（例：MDT地域でカメラがPDT設定のまま）

---

## 対応フォーマット

| 種別 | フォーマット |
|------|------------|
| 入力GPX | `.gpx`（GPX 1.1準拠、UTCタイムスタンプ） |
| 入力写真 | `.jpg` / `.jpeg` |
| タグ書き込み | ExifのGPS情報（exiftool使用） |

### 写真リストのクリック動作

1. 写真リストの行をクリック → その行をハイライト
2. 右ペイン下部に選択写真のプレビュー画像・EXIF詳細・マッチング結果を表示

## 機能仕様

### 1. GPXファイル読み込み
- **.gpxファイルをウィンドウにドラッグ&ドロップ**して読み込む（主要操作）
- ファイル選択ボタンからもダイアログで指定可能（補助操作）
- パース後にトラックポイント数と期間を表示
- 例：「624,906ポイント / 2026-04-08 〜 2026-04-18」

### 2. 写真フォルダ指定
- **デフォルト**: `/Users/junpeiwada/Documents/LightroomOutput`（設定で変更可能）
- 起動時にデフォルトフォルダを自動で読み込む
- フォルダ選択ボタンでダイアログから変更可能
- JPEG（`.jpg` / `.jpeg`）を再帰なしでリストアップ
- ファイル数を表示
- 設定変更はelectron-storeで永続化（次回起動時も反映）

### 3. タイムゾーン設定

#### 基本方針
- SONYカメラ（α1、α7RV等）は `OffsetTimeOriginal` を必ず記録するため、**原則として全ファイルを自動処理**する
- GPXのタイムスタンプはUTC（末尾Z）なので、**常にUTC同士でマッチング**する
- GPXファイル名に含まれるUTCオフセット情報（例：`Minus0700`）は使用しない

#### 自動モード（デフォルト・推奨）
- `OffsetTimeOriginal` をEXIFから取得してUTCに変換
- 例：`DateTimeOriginal: 2026:04:13 04:52:50` + `OffsetTimeOriginal: -08:00` → UTC `12:52:50`

#### 手動指定モード（フォールバック）
- `OffsetTimeOriginal` が存在しないファイルが含まれる場合の保険
- UTC-12〜UTC+14 のオフセットを選択し、全ファイルに一律適用
- `OffsetTimeOriginal` 未記録のファイルは自動モードでも手動指定へフォールバック

### 4. 写真プレビューパネル（右ペイン下部）
- 写真リストの行をクリックすると表示（未選択時は空欄 or プレースホルダ）
- **表示内容**:
  - 写真サムネイル（`<img>`タグ、`file://`パスで直接表示）
  - ファイル名
  - 撮影時刻（DateTimeOriginal + OffsetTimeOriginal）
  - UTC変換後の時刻
  - マッチしたGPXポイントの座標・高度
  - GPXポイントとの時間差（秒）
  - 状態（✓ 完了 / ⚠ 警告 / - 未処理）
- プレビュー実行前は「撮影時刻・ファイル名」のみ表示、実行後にマッチング結果を追加表示

### 5. マップ表示
- **地図ライブラリ**: Leaflet.js + OpenStreetMap（APIキー不要）
- GPXトラックを青線で表示
- 処理済み写真の座標を赤ピンで表示
- 写真リストの行をクリックすると、対応ピンを青くハイライトして地図をその位置に移動
- ピンをクリックするとポップアップでファイル名・撮影時刻・座標を表示
- タグ付与前後でピン位置が変わった場合、変更前（グレー）と変更後（赤）を両方表示

### 6. 標準ワークフロー

```
① GPXをD&D or 選択 → トラック表示
② 写真フォルダ読み込み（起動時自動）→ 写真リストに一覧表示
③「プレビュー」実行 → 全写真のマッチング結果を写真リストとマップに反映
④ 気になる写真をクリックして確認（プレビューパネル + マップのピン）
   → 問題なければ次へ、⚠が多ければGPXや設定を見直す
⑤「付与」実行 → ✓ 完了 のファイルを全件一括書き込み（⚠ 警告は自動スキップ）
```

### 7. プレビュー
- 実際には書き込まずに処理結果を写真リストとマップに反映
- 各ファイルの撮影時刻・UTC変換結果・マッチしたGPXポイントの座標・時間差を表示
- 「問題あり」（時間差が大きい等）のファイルを ⚠ 警告表示
- マップ上で任意のピンをクリック・写真リストの任意の行をクリックして個別確認可能

### 8. タグ付与
- 「付与」ボタンで ✓ 完了 ステータスの全ファイルを一括処理
- ⚠ 警告ステータスのファイルは自動スキップ（書き込まない）
- exiftoolで `-overwrite_original` によりGPS情報を上書き
- 完了後に「成功 N件 / スキップ M件」のサマリを表示

### 9. 上書き制御
- 「既存のGPSタグを上書き」チェックオフ時：既にGPS情報があるファイルをスキップ
- チェックオン時：既存のGPSタグを新しい座標で上書き

## マッチング処理ロジック

```
1. JPEGのDateTimeOriginalを取得（例: 2026:04:13 04:52:50）
2. OffsetTimeOriginalを取得（例: -08:00）
   - 存在しない場合は手動指定モードへフォールバック
3. UTCに変換（例: 04:52:50 - (-08:00) → 12:52:50 UTC）
4. GPXトラックポイント（すべてUTC）から最近傍ポイントをバイナリサーチで検索
5. 時間差が設定値（デフォルト3600秒）以内であればGPS座標を採用
6. exiftoolでGPSLatitude/Longitude/Altitudeを書き込む
```

### 実データによる検証例

| 項目 | 値 |
|------|------|
| DateTimeOriginal | 2026:04:13 04:52:50 |
| OffsetTimeOriginal | -08:00 |
| UTC変換後 | 2026-04-13T12:52:50Z |
| GPXマッチ座標 | 36°59'18.96"N, 110°6'47.16"W |
| 高度 | 1757.7m |

## 制約・注意事項

- exiftoolがインストールされていること（Homebrewで導入）
- GPXのタイムスタンプはUTC（末尾Z）であること
- 対応するGPXポイントが見つからない場合（時間差超過）はスキップ
- `OffsetTimeOriginal` がないJPEGは手動指定モードでのみ処理可能
- GPXファイル名に含まれるUTCオフセット（例：`Minus0700`）はマッチングに使用しない。カメラのEXIF値が正
