# GeoTagger 実装計画

**作成日**: 2026-04-23  
**対応仕様書**: `仕様.md` v0.1

---

## 全体方針

- フェーズ1→2→3の順に動く状態を積み上げる
- 各フェーズ末に単体で動作確認できるマイルストーンを置く
- 外部依存（exiftool・Leaflet）はフェーズ2以降で導入し、フェーズ1は純粋なElectron基盤に集中する

---

## フェーズ1：プロジェクト基盤 + ファイル読み込み

**ゴール**: アプリが起動し、GPXと写真フォルダを読み込んで一覧表示できる

### 1-1. プロジェクト初期化

- `package.json` 作成（Electron + electron-store + electron-builder）
- `main.js`・`preload.js`・`index.html` の最小構成
- `src/` ディレクトリ構成を確定

```
GeoTagger/
├── main.js              # Electronメインプロセス
├── preload.js           # IPC API定義
├── index.html           # UI本体
├── styles.css           # スタイル
├── src/
│   ├── gpx-parser.js    # GPXパース
│   ├── exif-reader.js   # EXIF読み取り（exiftool呼び出し）
│   ├── matcher.js       # UTC同士のマッチングロジック
│   ├── map-handler.js   # Leaflet操作
│   └── store.js         # electron-store ラッパー
└── package.json
```

### 1-2. 設定永続化（electron-store）

- デフォルト写真フォルダ `/Users/junpeiwada/Documents/LightroomOutput` を初期値として保存
- 起動時に保存済みフォルダパスを読み込む
- フォルダ変更時に即時保存

### 1-3. GPX読み込み

- ウィンドウ全体をD&Dターゲットにする（`dragover` / `drop` イベント）
- ファイル選択ボタン（補助）
- `src/gpx-parser.js`：DOMParserで `<trkpt>` を全件パース、UTC Dateオブジェクトの配列として保持
- パース後に「N,NNNポイント / YYYY-MM-DD 〜 YYYY-MM-DD」を表示

### 1-4. 写真フォルダ読み込み

- 起動時に保存済みフォルダを自動スキャン
- フォルダ選択ボタンでダイアログから変更可能
- `src/exif-reader.js`：exiftool を `child_process.execFile` で呼び出し、各JPEGから `DateTimeOriginal`・`OffsetTimeOriginal`・既存GPS有無を取得
- 写真リストに「ファイル名 / 撮影時刻+オフセット / 状態: 未処理」を表示

**マイルストーン①**: GPXを読み込むとポイント数が出て、写真リストにファイル名と撮影時刻が並ぶ

## フェーズ2：マッチングエンジン + プレビュー

**ゴール**: プレビューボタンで全写真のGPX座標マッチング結果が一覧に出る

### 2-1. マッチングロジック（src/matcher.js）

- `OffsetTimeOriginal` からUTCに変換
  - 存在しない場合は手動指定モードを要求（UI側でフォールバック表示）
- GPXポイント配列をソート済みとしてバイナリサーチで最近傍を検索
- 時間差が閾値（デフォルト3600秒）以内 → ステータス `✓ 完了`
- 超過 → ステータス `⚠ 警告`（座標は保持するが書き込まない）

### 2-2. プレビュー実行

- 「プレビュー」ボタン押下でメインプロセス経由にマッチング処理を渡す（IPC）
- 結果を写真リストの各行に反映（座標・時間差・状態を更新）
- 「既存GPSタグを上書き」オフの場合：既存GPS持ちファイルを `- スキップ` 扱い

### 2-3. タイムゾーン手動モードUI

- ラジオボタン「自動 / 手動」切り替え
- 手動選択時はUTCオフセットドロップダウン（UTC-12〜UTC+14）を表示
- `OffsetTimeOriginal` がないファイルが存在する場合に警告バナーを出す

**マイルストーン②**: プレビュー実行後、写真リストに座標・時間差・✓/⚠ が並ぶ

---

## フェーズ3：マップ表示 + 写真プレビューパネル

**ゴール**: マップ上にGPXトラックと写真ピンが出て、写真クリックで連動する

### 3-1. Leaflet.js 統合（src/map-handler.js）

- `index.html` にLeaflet CDN（またはローカルバンドル）を組み込む
- GPX読み込み後：トラックポイントを青ポリラインで描画、バウンズにフィット
- プレビュー後：✓ 完了 の写真を赤ピンで描画

### 3-2. 写真リスト ↔ マップ連動

- 写真リストの行クリック → 対応ピンを青にハイライト、マップをその座標に `panTo`（ズーム維持）
- マップのピンクリック → 対応行をハイライト、ポップアップでファイル名・撮影時刻・座標を表示

### 3-3. 写真プレビューパネル（右ペイン下部）

- 右ペインを上下にCSS Grid で分割（マップ上 / プレビュー下、境界ドラッグでリサイズ）
- 写真リスト行クリック時に表示：
  - `<img src="file://...">` でサムネイル
  - ファイル名・撮影時刻・OffsetTimeOriginal・UTC変換後時刻
  - マッチ座標・高度・時間差・状態（プレビュー前は撮影情報のみ）

**マイルストーン③**: マップ上にGPXトラックとピンが表示され、写真クリックで地図が動く

---

## フェーズ4：タグ付与 + 仕上げ

**ゴール**: 実際にexiftoolで書き込めて、アプリとして完成する

### 4-1. タグ付与実行

- 「付与」ボタン押下で確認ダイアログ（「✓ 完了 N件を書き込みます。よろしいですか？」）
- `✓ 完了` のファイルに対して exiftool を順次呼び出し
  - `-GPSLatitude`, `-GPSLongitude`, `-GPSAltitude`, `-GPSLatitudeRef`, `-GPSLongitudeRef`, `-GPSAltitudeRef`, `-overwrite_original`
- 進捗バーまたは処理中スピナーを表示
- 完了後サマリ：「成功 N件 / スキップ M件」

### 4-2. エラーハンドリング

- exiftool が見つからない場合：起動時チェックで警告バナー表示
- EXIF読み取り失敗ファイル：個別に `⚠ 読み取りエラー` 表示
- 書き込み失敗ファイル：個別にエラー状態を記録してサマリに含める

### 4-3. UI仕上げ

- ウィンドウサイズ・ペイン比率の保存（electron-store）
- GPX D&D中のビジュアルフィードバック（ハイライト枠）
- 処理中は「プレビュー」「付与」ボタンを無効化
- アプリアイコン設定

### 4-4. ビルド設定

- `electron-builder` の `build` セクションを `package.json` に設定
- macOS `.dmg` パッケージ生成確認

**マイルストーン④**: 実際の写真にGPS情報が書き込まれ、Lightroomで確認できる

## 依存パッケージ

| パッケージ | 用途 | 導入フェーズ |
|-----------|------|------------|
| `electron` | デスクトップアプリ基盤 | フェーズ1 |
| `electron-store` | 設定永続化 | フェーズ1 |
| `electron-builder` | macOS配布パッケージ生成 | フェーズ4 |
| `leaflet` | マップ表示（CDNまたはnpm） | フェーズ3 |

外部ツール: `exiftool`（Homebrew導入済み前提、起動時に存在確認）

---

## 実装順序サマリ

```
フェーズ1  基盤 + ファイル読み込み    → 写真リストに撮影時刻が並ぶ
フェーズ2  マッチングエンジン         → プレビューで座標・状態が出る
フェーズ3  マップ + プレビューパネル  → 地図連動・写真確認ができる
フェーズ4  タグ付与 + 仕上げ          → 実際に書き込める・配布できる
```
